# apps-baccarat-resource

## 项目简介

这是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或具体网站，仅作为一组静态资源的集中存放与版本管理空间。所有页面均为独立 HTML 文件，可直接在浏览器中打开或部署到静态托管服务。

## 目录结构说明

```
apps-baccarat-resource/
├── index.html          # 可选入口页面，提供资源导航
├── pages/              # 存放独立 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要归档目录，每个 HTML 文件为一个独立页面。
- `assets/`：存放页面间共享的样式、脚本、图片等静态资源。

## 页面归档说明

- 所有归档页面均为独立、自包含的 HTML 文件，尽量不依赖外部第三方资源。
- 页面命名建议使用英文小写字母、数字与连字符，便于 URL 直接访问。
- 如需引用公共资源，请使用相对路径指向 `assets/` 目录。

## 维护说明

- 新增页面请放入 `pages/` 目录，并在 `index.html`（如有）中添加导航链接。
- 更新现有页面时，建议保留旧版本作为历史记录（可通过 Git 提交历史回溯）。
- 公共资源文件请统一存放在 `assets/` 中，避免重复上传。
- 请勿在页面内容中包含任何具体域名、网址或营销性诱导文本。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或通过静态服务器部署整个仓库。
3. 如需自定义入口，可修改 `index.html` 文件。

---

本仓库仅用于静态 HTML 资源的归档与发布，不涉及任何后端服务或动态内容。
